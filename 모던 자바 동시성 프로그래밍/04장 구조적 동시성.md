# 구조적 동시성

<br>

## 1. 비구조적 동시성의 한계

- 하나의 부모 태스크가 여러개의 서브 태스크에 의존하고 있고, 각각의 서브 태스크가 서로 다른 스레드에서 실행될 때 부모 태스크가 예상치 못하게 종료된다면 여전히 서브 태스크는 독립적으로 실행을 계속하고 있음, 서브 태스크의 작업은 아무런 의미가 없음에도 불구하고 계속 실행되어 자원을 소모함
- `ExecutorService`나 `CompletableFuture`는 태스크의 **위계 구조**를 지원하지 않음

#### 예제

<details>
	<summary>예제 코드</summary>

```java
public class ProductService {

    public ProductInfo getProductInfo(Long productId) {
        try (var executorService = Executors.newVirtualThreadPerTaskExecutor()) {
            Future<Product> productTask = executorService.submit(() -> getProduct(productId));
            Future<List<Review>> reviewsTask = executorService.submit(() -> getReviews(productId));

            Product product = productTask.get();
            log("상품 조회");

            List<Review> reviews = reviewsTask.get();
            log("리뷰 조회");

            return new ProductInfo(product, reviews);
        } catch (ExecutionException | InterruptedException e) {
            throw new IllegalStateException(e);
        }
    }

    private Product getProduct(Long productId) {
        log("상품 조회 중...");
        sleep(Duration.ofSeconds(1));
        return new Product(productId, "IPhone", "good");
    }

    private List<Review> getReviews(Long productId) {
        log("리뷰 조회 중...");
        sleep(Duration.ofSeconds(2));
        return List.of(
                new Review(1L, "Excellent!!,", 5, productId),
                new Review(2L, "Good!!,", 4, productId)
        );
    }

    private void sleep(Duration duration) {
        try {
            Thread.sleep(duration);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new RuntimeException("Thread interrupted during sleep ", e);
        }
    }

    private static void log(String message) {
        Thread thread = Thread.currentThread();
        String threadName = thread.isVirtual() ? "VThread[#" + thread.threadId() + "]" : thread.getName();
        System.out.printf("%s %s %s%n", thread, threadName, message);
    }

    record Product(Long id, String name, String description) {

    }

    record Review(Long id, String comment, int rating, Long productId) {

    }

    record ProductInfo(Product product, List<Review> reviews) {

    }
}

public class Main {

    public static void main(String[] args) throws Exception {
        ProductService productService = new ProductService();
        productService.getProductInfo(1L);
    }
}
```

#### 📌 비구조적 동시성의 한계

- 비구조적 동시성은 태스크들을 서로 독립적인 존재로 취급하며, 그 사이의 관계나 의존성을 무시함 따라서 오류, 자원 누수, 성능 병목이 발생할 수 있으나 이러한 원인을 찾기는 어려움

## 2. 구조적 동시성

#### 기본 원칙

- 어떤 태스크가 여러개의 서브 태스크를 실행시킨다면 그 서브 태스크들은 모두 부모 태스크 코드의 동일한 지점으로 돌아와야 한다.

<br>

#### 이점

1. 관련된 태스크 집합 전체를 통합적으로 관리할 수 있음

    > 하나의 서브 태스크에 오류가 발생하거나 취소되면 같은 **위계 구조**에 있는 태스크들은 자동으로 종료되며, 오류는 태스크 위계 구조를 따라 상위로 전달됨

2. 스레드 누수 방지

    > 태스크의 부모-자식 생애주기를 엄격하게 보장함으로써 스레드 누수를 방지하고, 태스크는 자신의 유효 범위를 벗어나 존재할 수 없음

3. 관측 가능성을 높임

    > 부모-자식 관계가 명확하기 때문에 관측 가능성을 높이고 스레드 덤프나 프로파일러 도구에서 관계를 쉽게 추적할 수 있음

<br>

#### 가상 스레드와 구조적 동시성

- 각 요청을 가상 스레드에 할당하고, 가상 스레드 내부에서 여러개의 서브 태스크를 할당한다면 하나의 요청이 **트리 구조**를 이루고, 디버깅과 추적이 쉽지 않을까?

```txt
요청 1
 └─ 가상 스레드
     ├─ DB 호출 (가상 스레드)
     ├─ API 호출 (가상 스레드)
     └─ 캐시 조회 (가상 스레드)
```

<br>

```java
public class ProductService {

    public ProductInfo getProductInfo(Long productId) {
        Instant start = Instant.now();
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            StructuredTaskScope.Subtask<Product> productTask = scope.fork(() -> getProduct(productId));
            StructuredTaskScope.Subtask<List<Review>> reviewsTask = scope.fork(() -> getReviews(productId));
            scope.join();           // 모든 하위 작업이 끝날 때까지 대기
            scope.throwIfFailed();  // 하나라도 실패하면 예외 전파

            Product product = productTask.get();
            log("상품 조회");

            List<Review> reviews = reviewsTask.get();
            log("리뷰 조회");

            return new ProductInfo(product, reviews);
        } catch (ExecutionException | InterruptedException e) {
            throw new IllegalStateException(e);
        } finally {
            Instant end = Instant.now();
            System.out.println("total time: " + (end.toEpochMilli() - start.toEpochMilli()) + "ms");
        }
    }

    private Product getProduct(Long productId) {
        log("상품 조회 중...");

        if (productId == 1L) {
            throw new IllegalStateException("상품을 조회할 수 없습니다.");
        }

        sleep(Duration.ofSeconds(1));
        return new Product(productId, "IPhone", "good");
    }

    private List<Review> getReviews(Long productId) {
        log("리뷰 조회 중...");
        sleep(Duration.ofSeconds(2));
        return List.of(
                new Review(1L, "Excellent!!,", 5, productId),
                new Review(2L, "Good!!,", 4, productId)
        );
    }

    private void sleep(Duration duration) {
        try {
            Thread.sleep(duration);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new RuntimeException("Thread interrupted during sleep ", e);
        }
    }

    private static void log(String message) {
        Thread thread = Thread.currentThread();
        String threadName = thread.isVirtual() ? "VThread[#" + thread.threadId() + "]" : thread.getName();
        System.out.printf("%s %s %s%n", thread, threadName, message);
    }

    record Product(Long id, String name, String description) {

    }

    record Review(Long id, String comment, int rating, Long productId) {

    }

    record ProductInfo(Product product, List<Review> reviews) {

    }
}

public class Main {

    public static void main(String[] args) throws Exception {
        ProductService productService = new ProductService();
        productService.getProductInfo(1L);
    }
}
```

## 3. Joiner를 통한 여러 정책

- `Joiner`는 **join()** 메서드가 어떤 조건을 만족할 때 완료되는지, 어떤 결과를 반환할지를 정의함으로써 서브 태스크의 생애주기를 관리할 수 있는 매커니즘

<br>

### 3-1. awaitAllSuccessfulOrThrow

- 모든 서브 태스크가 성공해야 한다는 정책
- 서브 태스크 중 하나라도 실패하면 전체 스코프가 즉시 종료되며, 진행 중이던 다른 서브 태스크드른 즉시 취소되고, 실패한 서브 태스크의 예외가 부모로 전달됨
- join 메서드는 Void를 반환함
- 서브 태스크마다 서로 다른 타입의 결과를 반환할 때 사용하기 좋음

#### 예제

<details>
	<summary>예제 코드</summary>

```java
public class Main {

    public static void main(String[] args) throws Exception {
        Instant start = Instant.now();
        try (var scope = StructuredTaskScope.open(StructuredTaskScope.Joiner.<String>awaitAllSuccessfulOrThrow())) {
            var subtask1 = scope.fork(Main::getData1);
            var subtask2 = scope.fork(Main::getData2);
            var subtask3 = scope.fork(Main::getData3);
            var subtask4 = scope.fork(Main::getData4);

            scope.join();

            System.out.println("첫 번째 결과: " + subtask1.get());
            System.out.println("두 번째 결과: " + subtask2.get());
            System.out.println("세 번째 결과: " + subtask3.get());
            System.out.println("네 번째 결과: " + subtask4.get());
        } finally {
            Instant end = Instant.now();
            System.out.println((end.toEpochMilli() - start.toEpochMilli()) + "ms");
        }
    }

    private static String getData1() throws Exception {
        Thread.sleep(1000);
        return "data 1";
    }

    private static String getData2() throws Exception {
        Thread.sleep(2000);
        return "data 2";
    }

    private static String getData3() {
        throw new RuntimeException("data 3 error");
    }

    private static String getData4() {
        throw new RuntimeException("data 4 error");
    }
}
```

### 3-2. anySuccessfulResultOrThrow

- 여러 서브 태스크 중 가장 먼저 완료된 태스크를 선택함
- 가장 먼저 완료된 서브 태스크가 있으면 나머지 서브 태스크들은 **취소**됨

#### 예제

<details>
	<summary>예제 코드</summary>

```java
public class Main {

    public static void main(String[] args) throws Exception {
        Instant start = Instant.now();
        try (var scope = StructuredTaskScope.open(StructuredTaskScope.Joiner.<String>anySuccessfulResultOrThrow())) {
            scope.fork(Main::getData1);
            scope.fork(Main::getData2);
            scope.fork(Main::getData3);
            scope.fork(Main::getData4);

            String result = scope.join();

            System.out.println("가장 먼저 완료된 결과: " + result);
        } finally {
            Instant end = Instant.now();
            System.out.println((end.toEpochMilli() - start.toEpochMilli()) + "ms");
        }
    }

    private static String getData1() throws Exception {
        Thread.sleep(1000);
        return "data 1";
    }

    private static String getData2() throws Exception {
        Thread.sleep(1000);
        return "data 2";
    }

    private static String getData3() throws Exception {
        Thread.sleep(1000);
        return "data 3";
    }

    private static String getData4() throws Exception {
        Thread.sleep(1000);
        return "data 4";
    }
}
```

### 3-3. allSuccessfulOrThrow

- `allSuccessfulOrThrow`는 기본 정책인 **awaitAllSuccessfulOrThrow**와 마찬가지로 **전부 성공 아니면 실패** 전략을 따름
- 서브 태스크 중 하나라도 실패하면 즉시 다른 서브 태스크를 취소하고, `FailedException`예외가 발생함
- join 메서드는 결과값을 확인할 수 있음
- 결과는 `fork` 메서드를 호출한 순서대로 반환함
- 모든 서브 태스크가 동일한 반환 타입을 가질 때 이 정책 사용하면 좋음

#### 예제

<details>
	<summary>예제 코드</summary>

```java
public class Main {

    public static void main(String[] args) throws Exception {
        Instant start = Instant.now();
        StructuredTaskScope.open();
        try (var scope = StructuredTaskScope.open(StructuredTaskScope.Joiner.<String>allSuccessfulOrThrow())) {

            System.out.println("🚀 fork 시작");

            scope.fork(() -> getData("A", 3000)); // 가장 늦게 끝남
            scope.fork(() -> getData("B", 1000)); // 가장 빨리 끝남
            scope.fork(() -> getData("C", 2000));

            List<String> results = scope.join()
                    .map(StructuredTaskScope.Subtask::get)
                    .toList();

            System.out.println("모든 결과: " + results);
        } finally {
            Instant end = Instant.now();
            System.out.println((end.toEpochMilli() - start.toEpochMilli()) + "ms");
        }
    }

    private static String getData(String name, int sleepMs) throws Exception {
        System.out.println("👉 시작: " + name);
        Thread.sleep(sleepMs);
        System.out.println("✅ 완료: " + name);
        return name;
    }
}

// 결과

👉 시작: A
👉 시작: B
👉 시작: C
✅ 완료: B

✅ 완료: A
모든 결과: [A, B, C]

```
</details>

### 3-4. awaitAll

- `awaitAll`은 각 서브 태스크가 성공하든 실패하든 모두 완료될 때까지 기다림
- 일부 성공한 결과만이라도 처리해야하는 상황에 좋음

#### 예제

<details>
	<summary>예제 코드</summary>

```java
public class Main {

    public static void main(String[] args) throws Exception {
        Instant start = Instant.now();
        StructuredTaskScope.open();
        try (var scope = StructuredTaskScope.open(StructuredTaskScope.Joiner.<String>awaitAll())) {
            var subtask1 = scope.fork(Main::getData1);
            var subtask2 = scope.fork(Main::getData2);
            var subtask3 = scope.fork(Main::getData3);
            var subtask4 = scope.fork(Main::getData4);

            scope.join();

            printResult("subtask1", subtask1);
            printResult("subtask2", subtask2);
            printResult("subtask3", subtask3);
            printResult("subtask4", subtask4);
        } finally {
            Instant end = Instant.now();
            System.out.println((end.toEpochMilli() - start.toEpochMilli()) + "ms");
        }
    }

    private static void printResult(String name, StructuredTaskScope.Subtask<String> task) {
        switch (task.state()) {
            case SUCCESS -> System.out.println(name + " 성공: " + task.get());
            case FAILED -> System.out.println(name + " 실패: " + task.exception());
        }
    }

    private static String getData1() throws Exception {
        Thread.sleep(1000);
        return "data 1";
    }

    private static String getData2() throws Exception {
        Thread.sleep(1000);
        throw new RuntimeException("error data 2");
    }

    private static String getData3() throws Exception {
        Thread.sleep(1000);
        return "data 3";
    }

    private static String getData4() throws Exception {
        Thread.sleep(1000);
        throw new RuntimeException("error data 4");
    }
}

// 결과

subtask2 실패: java.lang.RuntimeException: error data 2
subtask3 성공: data 3

```

### 3-5. allUntil

- `allUntil`은 **Predicate**를 만족할 때까지 실행하고 멈춤
- 특정 조건에 만족되면 나머지 서브 태스크들은 작업이 취소됨

#### 예제

<details>
	<summary>예제 코드</summary>

```java
public class Main {

    private static final AtomicInteger SUCCESS_COUNT = new AtomicInteger();

    public static void main(String[] args) throws Exception {
        Instant start = Instant.now();
        StructuredTaskScope.open();
        try (var scope = StructuredTaskScope.open(StructuredTaskScope.Joiner.<String>allUntil(subtask -> {
                    if (subtask.state() == StructuredTaskScope.Subtask.State.SUCCESS) {
                        return SUCCESS_COUNT.incrementAndGet() >= 2;
                    }
                    return false;
            })
        )) {
            var subtask1 = scope.fork(Main::getData1);
            var subtask2 = scope.fork(Main::getData2);
            var subtask3 = scope.fork(Main::getData3);
            var subtask4 = scope.fork(Main::getData4);

            scope.join();

            printResult("subtask1", subtask1);
            printResult("subtask2", subtask2);
            printResult("subtask3", subtask3);
            printResult("subtask4", subtask4);
        } finally {
            Instant end = Instant.now();
            System.out.println((end.toEpochMilli() - start.toEpochMilli()) + "ms");
        }
    }

    private static void printResult(String name, StructuredTaskScope.Subtask<String> task) {
        switch (task.state()) {
            case SUCCESS -> System.out.println(name + " 성공: " + task.get());
            case FAILED -> System.out.println(name + " 실패: " + task.exception());
            case UNAVAILABLE -> System.out.println(name + " 이용불가: " + task.state());
        }
    }

    private static String getData1() throws Exception {
        Thread.sleep(1000);
        return "data 1";
    }

    private static String getData2() throws Exception {
        Thread.sleep(1000);
        throw new RuntimeException("error data 2");
    }

    private static String getData3() throws Exception {
        Thread.sleep(2000);
        return "data 3";
    }

    private static String getData4() throws Exception {
        Thread.sleep(3000);
        throw new RuntimeException("error data 4");
    }
}

// 결과
subtask1 성공: data 1
subtask2 실패: java.lang.RuntimeException: error data 2
subtask3 성공: data 3
subtask4 이용불가: UNAVAILABLE
2018ms
```
