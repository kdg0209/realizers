## 1. 긴 시퀀스 모델링의 문제점

- `트랜스포머`가 등장하기 전 [순환 신경망(RNN)](https://github.com/kdg0209/realizers/blob/main/LLM%EC%9D%84%20%ED%99%9C%EC%9A%A9%ED%95%9C%20%EC%8B%A4%EC%A0%84%20AI%20%EC%95%A0%ED%94%8C%EB%A6%AC%EC%BC%80%EC%9D%B4%EC%85%98%20%EA%B0%9C%EB%B0%9C/02%EC%9E%A5%20%ED%8A%B8%EB%9E%9C%EC%8A%A4%ED%8F%AC%EB%A8%B8%20%EC%95%84%ED%82%A4%ED%85%8D%EC%B2%98.md#1-rnn)이
언어 번역에 있어서 가장 인기 있는 **인코더-디코더** 구조였음
- RNN을 간단히 설명하면 이전 스텝의 출력이 다음 스텝의 입력으로 사용되는 신경망 구조여서 텍스트와 같은 순차 데이터에 잘 맞음

#### RNN의 가장 큰 문제점

- 인코더-디코더 RNN의 큰 제약 사항은 디코더 과정에서 RNN이 인코더의 이전 은닉 상태를 참조할 수 없음
- 결과적으로 디코드는 인코더로부터 받은 `최종 은닉 상태`에만 의존하므로 긴 문단에서는 맥락을 놓칠 수 있음
  - 특정 단어가 멀리 떨어진 단어에 의존성이 있는 복잡한 문장의 경우 제대로 판단할 수 없는 문제가 발생함
- 짧은 문장을 번역하는 것은 잘함

#### 바흐다나우 어텐션

- 이 매커니즘은 인코더-디코더 RNN을 수정하여 디코딩 단계마다 디코더가 선택적으로 입력 시퀀스의 서로 다른 부분을 참조 가능하게 함
- 현재 출력을 생성하는 데 어떤 입력 토큰이 다른 토큰보다 더 중요할 수 있기 때문에 중요도는 어텐션 가중치로 결정됨

<br>

## 2. 셀프 어텐션으로 입력의 서로 다른 부분에 주의 기울이기

#### 셀프 어텐션의 `셀프`란?

- 셀프 어텐션에서 `셀프`는 하나의 입력 시퀀스에 있는 서로 다른 위치의 원소 사이에서 **어텐션 가중치**를 계산하는 방식을 의미함
- 즉, 입력 자체에서 여러 부분 사이의 관계와 의존성을 평가하고 학습함

#### 셀프 어텐션의 목표

- 다른 모든 입력 정보를 조합하여 각각의 입력 정보에 대한 문맥 벡터를 계산하는 것
- 문맥 백터(z⑵) = 각 입력 정보의 중요도 또는 기여도는 어텐션 가중치 a₂₁ ~ a₂𝒕에 의해 결정됨

<br>

### 2-1. 간단한 셀프 어텐션 매커니즘

- 셀프 어텐션에서는 입력 시퀀스에 있는 각 원소 𝒙⁽¹⁾ ... 𝒙⁽ᵀ⁾에 대한 문맥 백터 z⁽𝔦⁾를 계산하는게 목표
- 문맥 백터의 목적은 문장과 같은 입력 시퀀스에 있는 다른 모든 원소의 정보를 통합해 이 시퀀스에 있는 각 원소의 표현을 풍부하게 만드는것
  - 문장에서 다른 단어 사이의 관계와 관련성을 이해하기 위해 LLM에 반드시 필요함
